Fileless Malware Simulation

Explored in-memory malware execution, persistence, and evasion techniques.

Developed a fileless malware that executed entirely in memory without writing to disk. Implemented persistence through Window Task Scheduler at SYSTEM privilegelevel and simulated malicious behaviors such as screenshot capture and command-and-control (C2) communication. Applied obfuscation techniques including AES encryption, Base64 encoding, and misleading script names to evade signature-based detection. This project highlighted the challenges defenders face with memory-only malware and emphasized the importance of behavior-based security monitoring

  • Python (C2 server, encryption)
  • PowerShell (payload & persistence)
  • Windows Task Scheduler
  • AES Encryption, Base64 Encoding

KRACK Attack Simulation

Demonstrated KRACK by replaying WPA2 handshake message 3 to force key reinstallation.

I built a controlled Wi-Fi lab to demonstrate the KRACK attack on WPA2 by replaying handshake Message-3 to force key reinstallation, which resets nonces and enables packet decryption/replay. The setup used a Kali attacker with a hostapd AP and TP-Link USB adapter (monitor/injection), Vanhoef’s KRACK PoC, and Wireshark for analysis; the victim was a Raspberry Pi (Raspbian 2016-05-13, wpa_supplicant 2.2). I replayed Message-3, captured the handshake and data frames, and verified nonce reuse and traffic exposure in Wireshark, confirming the vulnerable client state (“this is bad”) and demonstrating selective decryption/replay.

Mindwell

Developed a cognitive-distortion detection system using large language models and a custom multi-source dataset to advance mental health AI research.

MindWell is a research-driven project aimed at enhancing mental health support through the use of large language models (LLMs) such as GPT-4 and DeepSeek. Conducted as part of my B.Sc. Computer Science capstone at AUC, the project focused on building a comprehensive dataset and modeling pipeline to detect and classify 15 cognitive distortion types in written text. I contributed to multiple stages—including data collection from social media and LLM-generated text, annotation design, and model evaluation—while co-developing a Decision-of-Thought (DoT) framework to improve interpretability. Our models achieved ~76% accuracy, outperforming baselines (36–40%) and demonstrating potential for explainable, ethical AI in mental health contexts. The project culminated in a technical research paper documenting our dataset, methodology, and evaluation results.

  • Python . PyTorch · Hugging Face · NLP
  • Data collection & annotation · Model design & evaluation
  • Model evaluation
  • Research methods
